# Skill Maintenance Rule

AI skills and prompts are living workflow documents and MUST be updated whenever related:
- architecture
- standards
- commands
- workflows
- testing strategies
- verification processes
- Figma integration rules
- developer workflows
change.

Before completing a task, always evaluate whether:
- existing skills/prompts became outdated
- examples are still valid
- commands are still correct
- workflows remain synchronized with implementation

If not:
- update the affected skills/prompts
- update examples
- update related README references
- summarize skill changes in the final output
